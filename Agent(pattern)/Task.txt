Есть набор агентов, которые могут обрабатывать сообщения (как эти сообщения поступают, в 
общем случае не важно). При обработки сообщения разные типы агентов используют разные 
механизмы. Примем, что существуют 2 различных набора агентов. Обработка событий первым 
типом агентов заключается в выводе событий на консоль, обработка вторым типом – вывод 
события в список на форме. Разработать приложение, демонстрирующее работу такой системы. 
Какой использовать тип агентов – решается при загрузке приложения (информация хранится в
 конфигурационном файле). Использовать паттерны проектирования абстрактная фабрика – для
 порождения агентов, одиночка – для конкретных фабрик. Разработать 3-ий тип агентов, 
которые будут являться декораторами над 1-ым или 2-ым типом и кроме вывода сообщений, 
будут дополнительно записывать сообщения в файл. При реализации 3-его типа агентов и 
конкретной фабрики для их порождения использовать декораторы.
